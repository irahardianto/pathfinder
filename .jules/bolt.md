## $(date +%Y-%m-%d) - RipgrepScout Threading Overhead
**Learning:** In `crates/pathfinder-search`, the `RipgrepScout::search` implementation uses `MatchCollector` within a `for` loop over files. The state variables (`matches`, `total_count`, `skipped_count`) do not require synchronization primitives (`Mutex`) because the loop processes files strictly sequentially inside a single `spawn_blocking` task. The `Mutex` implementations add significant overhead via repeated un-contended locking and lock-poisoning detection per match and context buffer flush.
**Action:** Always verify if concurrency primitives (like `Mutex`) are truly needed based on the execution context. If work is done synchronously (e.g. within a simple `for` loop on a single thread), replace them with mutable references `&mut T` to eliminate overhead.
