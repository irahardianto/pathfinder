## 2026-05-22 - HashMap::entry allocation and Borrow Checker NLL Limitations
**Learning:** Replacing `.entry(key.clone()).or_insert(...)` with `if let Some(v) = map.get_mut(key) { v } else { map.insert(...); map.get_mut(...) }` avoids unconditional allocations on cache hits. However, if the result is assigned to a mutable reference variable and the map value is modified later in the same scope, the Rust borrow checker (due to NLL limitations) will throw E0499 ("cannot borrow `map` as mutable more than once at a time") because it considers the `get_mut` borrow in the `if let` condition to extend through the entire `else` block and beyond.
**Action:** When returning mutable references from conditional borrows on maps, be aware of this NLL limitation. If modifying the map within the `else` block, avoid holding the initial mutable borrow across the branch. Often it is better to perform the insertion conditionally and then retrieve the mutable reference unconditionally after the branch, e.g., `if !map.contains_key(key) { map.insert(key.clone(), ...); } let mut_ref = map.get_mut(key).unwrap();`.
