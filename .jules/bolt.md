## 2026-05-14 - Prevent Unconditional Allocations in HashMap::entry
**Learning:** Using `HashMap::entry(key.clone()).or_insert()` when the key needs to be cloned unconditionally causes memory allocations even on cache hits. In Rust, you can't simply replace this with `if let Some(v) = map.get_mut(&key)` and an `else` block containing `map.insert(key.clone(), ...)` because of NLL (Non-Lexical Lifetimes) borrow checker limitations: the mutable borrow from `get_mut` lasts through the `else` block.
**Action:** To avoid allocations on cache hits, use `if !map.contains_key(&key) { map.insert(key.clone(), ...); }` followed by `let value = map.get_mut(&key).expect("...");`. When the codebase forbids `expect` (e.g., via clippy's `-D warnings`), use `#[allow(clippy::expect_used)]` for the `expect` call, as we've safely proven the key exists.
