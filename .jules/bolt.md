## 2026-05-01 - Avoid String Clone in Ripgrep Searcher
**Learning:** In the hot path of the ripgrep searcher (`crates/pathfinder-search/src/ripgrep.rs`), `SearchMatch` structs were allocating a clone of the matched `content` string regardless of whether `context_lines` was requested. This caused an unnecessary allocation per match when `context_lines == 0`. We also discovered that `std::env::set_var("PATH")` used concurrently in tests causes flaky behavior with `which::which`.
**Action:** When conditionally buffering strings in structs across multiple branches, use `String::new()` as a non-allocating placeholder in struct initialization if the owned string is required, then assign or move the un-cloned `String` based on the condition. Always lock `PATH_MUTEX` when evaluating functions like `resolve_command` that depend on `which` in the LSP detection tests to prevent flaky tests.
