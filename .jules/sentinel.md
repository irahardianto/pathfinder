## 2025-05-25 - Prevent Git Argument Injection
**Vulnerability:** Found a command injection vulnerability in `crates/pathfinder-common/src/git.rs`. The `target` argument (e.g., a branch name or commit hash) was passed directly into `git diff --name-only <target>` using `tokio::process::Command`. Since git uses `--` to distinguish revisions from paths, an untrusted branch name starting with `-` could be interpreted as a command-line flag, potentially leading to argument injection or unexpected behavior.
**Learning:** `git` subprocess calls where the target parameter is untrusted input must not pass arguments starting with `-`. While normally we could append `--` before positional arguments to prevent this, git `diff` has a specific syntax (`git diff <tree-ish> [--] <path>...`) where `--` is specifically used to separate revisions from paths. If we passed `--`, `git` would interpret the user's `target` as a path instead of a revision, breaking functionality.
**Prevention:** Reject untrusted strings (e.g., git revisions) that start with a `-`.
