## 2024-05-19 - Prevent Git Subprocess Argument Injection
**Vulnerability:** The `git diff --name-only <target>` command in `SystemGit::diff_name_only` was vulnerable to argument injection. The `<target>` revision parameter, potentially sourced from untrusted inputs like `params.changed_since`, was appended directly to `Command::new("git")`. Since Git uses `--` to disambiguate revisions from paths, an attacker could supply a target starting with `-` (e.g., `--output=/tmp/pwned` or `--ext-cmd=touch /tmp/pwned`) to inject arbitrary flags, bypassing the intended command structure.
**Learning:** For git subprocess commands, `--` cannot safely terminate option parsing for revision parameters because Git treats it differently in that context. Thus, untrusted inputs intended as positional arguments for git revisions must be explicitly validated.
**Prevention:** Reject untrusted revision strings that begin with a hyphen (`-`) before passing them to the subprocess command to prevent argument injection.
